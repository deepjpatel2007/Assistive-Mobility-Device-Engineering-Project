# Assistive Mobility Device Engineering Project

## Arduino-Powered Teddy Bear Wheelchair / “Ambearlance”

This repository showcases the **Teddy Bear Wheelchair Project**, an ENGG*1100 engineering design project where our team designed, built, tested, and documented an Arduino-powered miniature assistive mobility vehicle. The final prototype, nicknamed the **Ambearlance**, combined mechanical design, electronics, embedded control, safety testing, cost analysis, and engineering documentation.

The purpose of this repository is to present the project in a clean professional format while keeping the original school deliverables and extracted project data organized.

## Project Overview

The goal of the project was to create a small-scale wheelchair-style mobility device capable of safely transporting a teddy bear while completing course-based performance tasks. The final design used a custom mechanical chassis, Arduino-based control logic, a motor-driven drivetrain, a servo-triggered launcher, and an ambulance-themed exterior shell.

The project demonstrates a complete engineering design process, including concept generation, subsystem design, prototyping, testing, analysis, and reflection.

---

## Highlights

- Built an Arduino-powered mobility prototype from mechanical and electrical components
- Designed a three-wheel chassis selected through weighted decision analysis
- Integrated a DC motor, motor controller, Arduino Uno, battery system, and wiring
- Created a servo-triggered mousetrap launcher for the Eco-Trash Launch event
- Achieved a **5.81 second** speed run and a **30/30** speed score
- Achieved **26/30** on the Eco-Trash Launch event
- Passed required safety and stability tests
- Documented cost, performance, lifecycle impact, centre of mass, and design tradeoffs

## What This Repository Contains

This repo is organized into several sections:

- **`docs/`** – professional project documentation, including design process, decisions, testing, safety, lifecycle analysis, team contributions, and lessons learned
- **`data/`** – extracted project data such as the bill of materials, performance results, decision matrices, centre of mass values, and lifecycle metrics
- **`hardware/`** – subsystem documentation for the mechanical design, electrical system, launcher system, and overall architecture
- **`software/`** – Arduino control logic documentation
- **`images/`** – prototype, final design, development, and testing photos
