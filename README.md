# Assistive Mobility Device Engineering Project

## Overview

The **Assistive Mobility Device Engineering Project** is an engineering design project focused on creating a functional mobility-support prototype using mechanical design, electronics, and embedded systems. The project was developed to explore how assistive technology can improve movement, accessibility, and user support through an integrated hardware and software solution.

This project combines a custom chassis, Arduino-based control, servo motor integration, and iterative testing to build a prototype that demonstrates core principles of assistive mobility design.

## Project Purpose

The main goal of this project was to design and develop an assistive mobility device prototype that could support movement-related tasks while applying engineering design principles. The project emphasized:

- Improving accessibility through practical assistive technology
- Integrating mechanical, electrical, and software components
- Designing a stable and functional prototype structure
- Testing and refining the system through multiple design iterations
- Applying problem-solving, teamwork, and engineering documentation skills

## Key Features

- Arduino-controlled servo motor system
- Custom chassis design for mobility support
- Mechanical and electronic system integration
- Prototype testing and troubleshooting
- Iterative design improvements based on testing results
- Focus on accessibility, stability, and usability

## Technologies and Tools Used

- **Arduino** – microcontroller programming and hardware control

## Design Process

The project followed a structured engineering design process:

1. **Problem Identification**  
   The team identified the need for an assistive mobility concept that could support movement and accessibility.

2. **Research and Planning**  
   Possible design solutions were explored, including chassis structure, movement control, and electronic integration.

3. **Prototype Design**  
   A prototype structure was developed using CAD and physical modelling techniques.

4. **System Integration**  
   Mechanical components, Arduino control, servo motors, and wiring were combined into one working prototype.

5. **Testing and Troubleshooting**  
   The device was tested to evaluate movement, stability, and component response. Issues were identified and improved through redesign and debugging.

6. **Final Refinement**  
   The prototype was adjusted to improve reliability, usability, and overall functionality.

## Challenges Faced

- Balancing design complexity with available materials and time
